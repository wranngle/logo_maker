# Contributing to logo_maker

Keep changes small, reproducible, and covered by the local check.

## Getting Started

1. Fork the repo and clone it locally.
2. Run `bun install`.
3. Create a branch for your feature or bug fix.

## Testing & Style

- Run `bun run check` before opening a PR.
- Add or update tests for behavior changes in `src/`.
- Do not commit generated `output/` files, local logs, or raw scratch images.

## Submitting a PR

- Keep your Pull Request focused on a specific feature or bug.
- Include a clear description of what changed and why.
- Link any relevant issues.
